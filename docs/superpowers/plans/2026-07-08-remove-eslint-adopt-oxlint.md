# Remove ESLint, Adopt Root oxlint — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Remove all ESLint dependencies from the ZipShip monorepo and make the existing root `oxlint` the single linter, invocable via `bun run lint` / `bun turbo run lint`.

**Architecture:** Single root `.oxlintrc.json` already exists and stays unchanged. Work is: (1) delete 4 ESLint entries from the root `package.json` catalog and move `oxlint` into `devDependencies`, (2) add `lint`/`lint:fix` scripts + `//#lint`/`//#lint:fix` Turborepo Root Tasks, (3) run oxlint, apply `--fix`, report anything left.

**Tech Stack:** Bun workspaces + Bun catalogs, Turborepo Root Tasks, oxlint 1.73.x.

## Global Constraints

- **Never use `git add -A` / `git add .`.** The working tree currently contains unrelated uncommitted changes (console-app UI, desktop-shell deletions, etc.). Every commit step stages **explicit file paths only** so this work is not commingled with pre-existing changes.
- oxlint version stays `^1.73.0` (do not bump).
- Single root `.oxlintrc.json` — do **not** create per-package configs. If a path needs different rules later, use an `overrides` block in the root config.
- Do **not** manually edit business/source code. Only oxlint's own `--fix` may touch source, and only in Task 3.
- Conventional-commit prefixes match the repo's style (`chore:`, `docs:`, etc.). End every commit message with a blank line then `Co-Authored-By: Claude <noreply@anthropic.com>`.

## Prerequisites

- Repo root is `g:\zipship`. Commands assume CWD = repo root.
- `bun` is available (the project pins `packageManager: bun@1.3.14`).
- The Bash tool (Git Bash) is used for verification commands; all commands are POSIX-compatible.

## Task 2: Add lint scripts and Turborepo Root Tasks

**Files:**
- Modify: `package.json` (scripts)
- Modify: `turbo.json` (tasks)

**Interfaces:**
- Consumes: oxlint installed (Task 1).
- Produces: `bun run lint`, `bun run lint:fix`, and `bun turbo run lint` / `bun turbo run lint:fix` all runnable from the repo root.

- [ ] **Step 1: Add `lint` and `lint:fix` scripts to package.json**

In `package.json`, the `"scripts"` block begins:
```json
  "scripts": {
    "typecheck": "tsc -p tsconfig.json",
    "typecheck:workspaces": "bun turbo run typecheck",
    "dev": "bun turbo run dev --filter=@zipship/api --filter=@zipship/web-shell",
```
Insert the two lint scripts immediately after `typecheck:workspaces` (grouped with the other quality checks):
```json
    "typecheck:workspaces": "bun turbo run typecheck",
    "lint": "oxlint .",
    "lint:fix": "oxlint --fix .",
    "dev": "bun turbo run dev --filter=@zipship/api --filter=@zipship/web-shell",
```

- [ ] **Step 2: Add `//#lint` and `//#lint:fix` Root Tasks to turbo.json**

`turbo.json` currently is:
```json
{
  "$schema": "https://turbo.build/schema.json",
  "globalDependencies": [".env"],
  "tasks": {
    "typecheck": {
      "dependsOn": [],
      "outputs": [],
      "inputs": ["src/**/*.ts", "src/**/*.tsx"]
    },
    "dev": {
      "cache": false,
      "persistent": true
    }
  }
}
```
Add the two Root Tasks inside `"tasks"` (after the `dev` task). The `//#` prefix registers them as root-workspace tasks that run the root `package.json` scripts. Result:
```json
{
  "$schema": "https://turbo.build/schema.json",
  "globalDependencies": [".env"],
  "tasks": {
    "typecheck": {
      "dependsOn": [],
      "outputs": [],
      "inputs": ["src/**/*.ts", "src/**/*.tsx"]
    },
    "dev": {
      "cache": false,
      "persistent": true
    },
    "//#lint": {},
    "//#lint:fix": {
      "cache": false
    }
  }
}
```
Note: `lint:fix` mutates files, so it MUST be `"cache": false`.

- [ ] **Step 3: Verify both JSON files parse**

Run: `bun --eval "JSON.parse(require('fs').readFileSync('turbo.json','utf8')); JSON.parse(require('fs').readFileSync('package.json','utf8')); console.log('JSON OK')"`
Expected: prints `JSON OK` (exit 0).

- [ ] **Step 4: Verify `bun run lint` works**

If it errors with `oxlint: command not found` / resolution failure, Task 1's install did not complete — go back.

- [ ] **Step 5: Verify `bun turbo run lint` works**

Run: `bun turbo run lint`
Expected: turbo prints its task banner (`• lint` / `> //#:lint`) and runs oxlint once at the root, then reports success or the oxlint error count. Confirms the Root Task is wired.

- [ ] **Step 6: Verify `lint:fix` is registered as a Root Task (no-op check only)**

Expected: prints the matched line and `OK: lint:fix root task present`.

- [ ] **Step 7: Commit**

```bash

git commit -m "chore: add oxlint lint scripts and turbo root tasks" -m "Add bun run lint / lint:fix at root and register //#lint (cacheable) and //#lint:fix (cache:false) as Turborepo Root Tasks per the official oxlint guide." -m "Co-Authored-By: Claude <noreply@anthropic.com>"
```

## Task 3: Run oxlint, apply auto-fixes, report the rest

**Files:**
- Possibly modify: any source file oxlint `--fix` rewrites (discovered at runtime).
- Does NOT modify: `.oxlintrc.json`, business logic (manual edits forbidden here).

**Interfaces:**
- Consumes: `bun run lint` / `bun run lint:fix` from Task 2.
- Produces: a clean (post-fix) lint state, a commit of the auto-fixed files, and a written report of any remaining non-fixable findings for the user.

- [ ] **Step 1: Capture the pre-fix baseline**

Run: `bun run lint 2>&1 | tee /tmp/oxlint-before.txt; echo "exit=$?"`
Expected: oxlint summary with a warning/error count. Save the count for comparison. (Non-zero exit is normal here if existing code has findings.)

- [ ] **Step 2: Apply only auto-fixable changes**

Run: `bun run lint:fix`
Expected: oxlint rewrites files in place for everything `--fix` can handle, then prints the remaining (non-fixable) count.

- [ ] **Step 3: Re-run lint to confirm the post-fix state**

Run: `bun run lint 2>&1 | tee /tmp/oxlint-after.txt; echo "exit=$?"`
Expected: the warning/error count is **lower than or equal to** Step 1. If it is unchanged, `--fix` had nothing to apply (fine). If it is *higher*, stop and investigate — a fix likely conflicted with existing code.

- [ ] **Step 4: Inspect exactly what `--fix` changed**

Run: `git status --short`
Expected: a list of modified source files. **Cross-check** that every listed file is one oxlint rewrote — none of these paths should be `package.json`/`turbo.json`/`bun.lock` (those are already committed in Tasks 1–2). If a file in the list was *already* modified before this task (pre-existing unrelated change), note it: its diff will contain both the pre-existing change and the lint fix, which is acceptable but should be called out in the report.

Run: `git diff -- <each changed path>` to eyeball that the diffs are mechanical lint fixes (import ordering, quote style, unused-var removal, trailing commas, etc.).

- [ ] **Step 5: Commit the auto-fixed files (explicit paths only)**

Stage **only** the files oxlint changed — take the exact paths from Step 4's `git status` output. Example (replace the paths with the real ones):
```bash
git add packages/console-app/src/foo.ts apps/api/src/bar.ts
git commit -m "style: apply oxlint --fix auto-fixes" -m "Mechanical fixes from 'bun run lint:fix' (import order, unused vars, etc.). No logic changes." -m "Co-Authored-By: Claude <noreply@anthropic.com>"
```
If oxlint changed nothing, skip this commit and note "no auto-fixes needed" in the report.

- [ ] **Step 6: Write the remaining-findings report for the user**

From `/tmp/oxlint-after.txt`, extract every remaining warning/error. Produce a short report with:
1. Total remaining count (warnings / errors).
2. Breakdown by rule (e.g. `react/only-export-components: 5`, `no-unused-vars: 2`).
3. For each rule, 1–2 example `file:line` locations.
4. A recommended next step per cluster: (a) fix manually, (b) relax/`overrides` the rule in `.oxlintrc.json`, or (c) accept as-is.

Present this report to the user in the session. **Do not** edit `.oxlintrc.json` or any source file manually in response to these findings without explicit user sign-off (per spec §4).

- [ ] **Step 7: Final verification of all done-criteria**

Run each and confirm:
```bash
# (a) no eslint references remain in tracked source/config (transitive deps may still appear under node_modules/ — ignore those)
git grep -niI "eslint" -- ':!bun.lock' ':!node_modules' ':!docs/superpowers' || echo "OK: no eslint refs in tracked files"
# (b) oxlint is the devDependency linter
grep -n '"oxlint"' package.json
# (c) root script + turbo task both run
bun run lint
bun turbo run lint
```
Expected:
- (a) prints `OK: no eslint refs in tracked files`.
- (b) shows `"oxlint": "^1.73.0",` under `devDependencies`.
- (c) both commands run oxlint successfully.

If all pass, the plan is complete. (There is no separate "Task 3 commit" beyond Step 5; final verification is read-only.)
